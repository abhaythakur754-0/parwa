# SSL/TLS Certificates for PARWA

This directory contains SSL/TLS certificates for production deployment.

## Directory Structure

```
nginx/ssl/
├── fullchain.pem    # Full certificate chain (server cert + intermediates)
├── privkey.pem      # Private key (chmod 600)
├── chain.pem        # Intermediate certificate chain (optional)
├── cert.pem         # Server certificate only (optional)
└── README.md        # This file
```

## Let's Encrypt Setup (Recommended for Production)

### Prerequisites

1. **Domain must point to server** - Ensure DNS A/AAAA records for `parwa.ai` and `www.parwa.ai` point to your server's IP.

2. **Ports 80 and 443 must be open** - Let's Encrypt performs HTTP-01 challenge on port 80.

3. **Nginx must be running** - The server block on port 80 must handle ACME challenges.

### Initial Certificate Request

Using Certbot (recommended):

```bash
# Install certbot
sudo apt-get update
sudo apt-get install -y certbot python3-certbot-nginx

# Request certificate (dry run first)
sudo certbot certonly --nginx --dry-run -d parwa.ai -d www.parwa.ai

# Request actual certificate
sudo certbot certonly --nginx -d parwa.ai -d www.parwa.ai

# Copy certificates to this directory
sudo cp /etc/letsencrypt/live/parwa.ai/fullchain.pem ./fullchain.pem
sudo cp /etc/letsencrypt/live/parwa.ai/privkey.pem ./privkey.pem
sudo chown $USER:$USER *.pem
chmod 644 fullchain.pem
chmod 600 privkey.pem
```

### Certificate Renewal

Let's Encrypt certificates are valid for 90 days. Set up automatic renewal:

```bash
# Test renewal
sudo certbot renew --dry-run

# Add to crontab (runs twice daily)
sudo crontab -e

# Add this line:
0 0,12 * * * certbot renew --quiet --post-hook "cp /etc/letsencrypt/live/parwa.ai/fullchain.pem /path/to/parwa/nginx/ssl/fullchain.pem && cp /etc/letsencrypt/live/parwa.ai/privkey.pem /path/to/parwa/nginx/ssl/privkey.pem && docker kill -s HUP parwa-nginx"
```

### Docker Deployment

When using Docker, mount the SSL directory:

```yaml
# docker-compose.prod.yml
services:
  nginx:
    image: nginx:alpine
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/ssl:/etc/nginx/ssl:ro
    ports:
      - "80:80"
      - "443:443"
```

### Manual Certificate Installation

If not using Let's Encrypt, place your certificates:

1. **fullchain.pem** - Your server certificate followed by intermediate certificates
2. **privkey.pem** - Your private key (must not have a passphrase)

**Important:** Never commit private keys to the repository. Add to `.gitignore`:

```gitignore
nginx/ssl/*.pem
nginx/ssl/*.key
!nginx/ssl/README.md
```

## Security Checklist

- [ ] Private key has no passphrase
- [ ] Private key has 600 permissions (owner read/write only)
- [ ] Certificate chain is complete (includes intermediates)
- [ ] Certificates are not expired
- [ ] Auto-renewal is configured
- [ ] Certificates cover all domains (parwa.ai, www.parwa.ai)
- [ ] TLS 1.2+ only (no SSLv3, TLS 1.0, TLS 1.1)
- [ ] Strong cipher suites configured in nginx.conf

## Verify Configuration

```bash
# Check certificate validity
openssl x509 -in fullchain.pem -text -noout

# Verify key matches certificate
openssl x509 -noout -modulus -in fullchain.pem | openssl md5
openssl rsa -noout -modulus -in privkey.pem | openssl md5

# Test SSL configuration
openssl s_client -connect parwa.ai:443 -servername parwa.ai

# Online test (run after deployment)
# https://www.ssllabs.com/ssltest/analyze.html?d=parwa.ai
```

## Troubleshooting

### Certificate Not Trusted

1. Ensure fullchain.pem includes intermediate certificates
2. Check certificate covers the exact domain being accessed
3. Verify system time is correct

### Nginx SSL Errors

```bash
# Test nginx configuration
nginx -t

# Check nginx error logs
docker logs parwa-nginx --tail 50
```

### Renewal Failures

1. Check port 80 is accessible from internet
2. Verify DNS records are correct
3. Check certbot logs: `/var/log/letsencrypt/letsencrypt.log`
